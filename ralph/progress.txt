- Vitest tests must go in `server/__tests__/**/*.test.ts` or `shared/__tests__/**/*.test.ts` ‚Äî NOT `server/tests/`

- Project uses ESM (`"type": "module"`) with `import.meta.url` for `__dirname` equivalent
- Path aliases: `@/*` ‚Üí `./client/src/*`, `@shared/*` ‚Üí `./shared/*`
- Server services live under `server/services/` as flat files or subdirectories
- Shared types go in `shared/` for cross-boundary use (client + server)
- Dependencies installed via npm (not bun, despite bun.lock presence)
- Use `Array.from(map.keys())` not `[...map.keys()]` ‚Äî TS target lacks downlevelIteration
- Use dependency injection (optional `storageOverride` param) for testable server services ‚Äî avoids MongoDB connection in tests
- Browsers can't use native gRPC; use SSE HTTP bridge (`server/services/conversation/http-bridge.ts`) for client ‚Üî gRPC conversation pipeline
