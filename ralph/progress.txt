## Codebase Patterns
- Typecheck: `npx tsc --noEmit` — pre-existing errors in server/tests/ and server/services/quest-*.ts are known; new files must not add errors
- Vitest tests must go in `server/__tests__/**/*.test.ts` or `shared/__tests__/**/*.test.ts` — NOT `server/tests/`
- Dynamic `import()` side effects (auto-registration) only fire once per process — after `clearProviders()` in tests, re-register manually
- Project uses ESM (`"type": "module"`) with `import.meta.url` for `__dirname` equivalent
- Path aliases: `@/*` → `./client/src/*`, `@shared/*` → `./shared/*`
- Server services live under `server/services/` as flat files or subdirectories
- Shared types go in `shared/` for cross-boundary use (client + server)
- Dependencies installed via npm (not bun, despite bun.lock presence)
- Use `Array.from(map.keys())` not `[...map.keys()]` — TS target lacks downlevelIteration
- Use dependency injection (optional `storageOverride` param) for testable server services — avoids MongoDB connection in tests
- Browsers can't use native gRPC; use SSE HTTP bridge (`server/services/conversation/http-bridge.ts`) for client ↔ gRPC conversation pipeline
- BabylonChatPanel uses graceful degradation: try gRPC streaming → fall back to direct Gemini API if unavailable
