## Codebase Patterns
- Street generator methods follow pattern: `generateX(config: StreetGenConfig & { extraFields }): StreetNetwork`
- Tests use custom assert/counter pattern (not a test framework): `npx tsx file.test.ts`
- GeographyConfig is defined in `server/generators/geography-generator.ts` and contains terrain, settlementType, population, foundedYear
- Pre-existing typecheck errors in BabylonGame files are expected; check only your changed files for errors
- GeographyGenerator now uses StreetGenerator internally; the `streets` field in settlement DB is a StreetNetwork object (not Location[])
- To detect old vs new street format: `Array.isArray(streets)` = old, `'nodes' in streets` = new StreetNetwork
- Use Bresenham-style fractional accumulator to distribute counts across many items (avoids rounding-to-zero with small fractions)
- `interpolateAtDistance(waypoints, d)` in lot-generator.ts uses absolute distance (not normalized t) — better for absolute spacing
- TypeScript target requires `Array.from()` for iterating Sets/Maps — never use `for..of` directly on Map/Set
- When modifying `shared/game-engine/types.ts` or `ir-types.ts`, run `npm run drift:update` before committing — pre-commit hook blocks otherwise

## 2026-03-12 - US-018
- Implemented `generateLotsAlongStreets(network: StreetNetwork, config: GeographyConfig): LotPosition[]` in new `lot-generator.ts`
- LotPosition type includes: position (Vec3), facingAngle, width, depth, side (left/right), distanceAlongStreet, streetEdgeId
- Lot width by settlement type: village=15, town=12, city=10
- Bresenham-style fractional accumulator distributes lots proportionally across edges by density-weighted capacity
- Lots placed on both sides of each street edge with perpendicular setback (street width/2 + lot depth/2)
- Facing angle computed as atan2 pointing from lot toward street center
- Guaranteed at least 1 lot when population > 0 and edges exist
- Files changed: `server/generators/lot-generator.ts` (new), `server/generators/lot-generator.test.ts` (new)
- **Learnings for future iterations:**
  - Bresenham accumulator pattern works well for distributing fractional counts across many edges — avoids rounding-to-zero problem with simple `Math.round(fraction)`
  - `interpolateAtDistance` (absolute distance) is more useful than normalized `t ∈ [0,1]` for lot placement since lot widths are absolute
  - `lotDensityByStreetType` is separate from `buildingDensityByStreetType` in geography-generator — alleys get 0.15 (almost no lots) vs 0.1 (almost no buildings)
  - Tangent perpendicular convention: left = (nz, -nx), right = (-nz, nx) — consistent with geography-generator
  - All 30 lot tests + 2,859 street tests pass; typecheck clean on modified files

## 2026-03-13 - US-020
- Implemented `generateBlocks(network: StreetNetwork, centerPoint?: Vec2): Block[]` in new `server/generators/block-generator.ts`
- Block interface: id, boundaryStreetIds, polygon (Vec2[]), districtId, blockNumber, center (Vec3)
- Uses planar face extraction via half-edge traversal: builds sorted adjacency lists by angle, then traces minimal faces using "next CW edge after reverse" algorithm
- Outer (unbounded) face filtered by signed area: inner faces have positive signed area (CCW winding), outer face has negative (CW)
- Block numbers assigned by sorting blocks by centroid distance from settlement center (100, 200, 300...)
- `assignLotsToBlocks(lots, blocks)` uses ray-casting point-in-polygon to assign each lot to its containing block
- Added `Block` type to `shared/game-engine/types.ts` and `BlockIR` to `shared/game-engine/ir-types.ts`
- Files changed: `server/generators/block-generator.ts` (new), `server/generators/block-generator.test.ts` (new), `shared/game-engine/types.ts`, `shared/game-engine/ir-types.ts`, `shared/game-engine/drift-manifest.json`
- **Learnings for future iterations:**
  - Planar face extraction uses half-edge traversal: for each directed edge u→v, the next edge in the face is the one just CW after the reverse direction v→u in v's angle-sorted adjacency list
  - Outer face detection: inner faces have positive signed area (CCW), outer face has negative (CW) — filter by sign, not magnitude
