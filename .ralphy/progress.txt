## Generic Skill Tree Framework

- Created `shared/skill-tree/index.ts` with generic parameterized types:
  - `SkillNode<T>`, `SkillTier<T>`, `SkillCondition<T>`, `SkillTreeState<T>`, `SkillTreeConfig<T>`
  - `createSkillTreeState<T>()` — creates fresh state from config
  - `updateSkillProgress<T>()` — updates progress and returns newly unlocked nodes
- Refactored `shared/language-skill-tree.ts` to use generic framework:
  - Defined `LanguageConditionType` union and `LanguageSkillStats` interface
  - Wired language stat resolver mapping condition types to stat fields
  - Maintained backward-compatible exports: `SkillNode`, `SkillTier`, `SkillTreeState`, `createDefaultSkillTreeState`, `updateSkillProgress`, `SKILL_TIERS`
  - Renamed `fluencyRange` to generic `range` field (updated BabylonSkillTreePanel.ts reference)
- Added `shared/__tests__/skill-tree.test.ts` with 11 tests covering both generic and language-specific behavior
