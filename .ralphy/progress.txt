## Conversation Context Caching on Server

- Created `server/services/conversation-context-cache.ts` — LRU cache with TTL for conversation contexts:
  - Configurable max size (default 200), TTL (default 30 min), max messages per conversation (default 50)
  - LRU eviction: least-recently-used entries removed when cache reaches capacity
  - TTL expiration: entries auto-expire after inactivity
  - Static key builders: `chatKey(worldId, npcId, playerId)` and `languageKey(languageId)`
  - Supports append (incremental message addition), set (full replace), get, delete, clear
  - Singleton `conversationContextCache` exported for server-wide use
- Integrated into Gemini chat endpoint (`server/routes.ts`):
  - When worldId + npcId + playerId are provided, caches conversation history server-side
  - On cache hit with matching systemPrompt, uses cached history instead of requiring client to resend full history
  - After successful response, appends both user message and model response to cache
- Integrated into language chat service (`server/services/language-service.ts`):
  - Caches MongoDB query results for message history
  - On cache hit, skips DB query and reconstructs history from cache
  - On cache miss, queries DB and populates cache for future requests
  - Updates cache incrementally on each new user/assistant message
